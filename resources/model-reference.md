# Model Reference

**Model Reference** is an index of every 3D model file in the game, identified by its model ID and file path.

**Credits:** Xeavin, Eochaid

### What can you do with this?

Backgrounds, party members, NPCs, monsters, summons, treasures, and weapons all have their own model entries here, each pointing to the actual `.cm` file inside the game's archive. Use this when you need to find or swap a specific model asset by name across the whole game. It covers both the standard and higher-poly cutscene variants where they exist, and notes model quirks like palette-swapped enemy variants or unused "Reserve" IDs that crash the game if referenced.

This is a global index, every model in the game, keyed by its raw `.cm` file. It's a different scope than the Enemy & Area Data Map's own models tab, which only lists the subset of models actually used within one specific area file. Use this sheet to find a model by name across the whole game, and the Enemy & Area Data Map when you need to know which of those models a given area actually references.
